User API Endpoints Documentation

1. Register a new user

  • Endpoint: POST /api/register
  • Request Body: { "email": "example@example.com", "password": "password123" }

2. Login with existing credentials

  • Endpoint: POST /api/login
  • Request Body: { "email": "example@example.com", "password": "password123" }
  • Response Body (on successful login): { "token": "<JWT_TOKEN>" }

3. Logout (invalidate the JWT token)

  • Endpoint: POST /api/logout
  • Request Body: None
  • Note: The client should handle removing the token from local storage or cookies.

4. Get user profile (requires authentication)

  • Endpoint: GET /api/profile
  • Request Headers: { "Authorization": "Bearer <JWT_TOKEN>" }
  • Response Body: { "user": { "_id": "<USER_ID>", "email": "example@example.com" } }

User Endpoints Documentation

Fetch All Users

  • URL: /api/users/
  • Method: GET
  • Description: Fetches a list of all users.
  • Response:
    • 200 OK on success
    • JSON array containing user objects

Fetch User by ID

  • URL: /api/users/:id
  • Method: GET
  • Description: Fetches a single user by their ID.
  • URL Parameters: id: The ID of the user to fetch.
  • Response:
    • 200 OK on success
    • JSON object representing the user
    • 404 Not Found if user with the specified ID is not found

Create New User

  • URL: /api/users/
  • Method: POST
  • Description: Creates a new user.
  • Request Body:
    • email: The email of the new user.
    • password: The password of the new user.
  • Response:
    • 201 Created on success
    • JSON object representing the newly created user

Update User

  • URL: /api/users/:id
  • Method: PUT
  • Description: Updates an existing user.
  • URL Parameters: id: The ID of the user to update.
  • Request Body:
    • email: The updated email of the user.
    • password: The updated password of the user.
  • Response:
    • 200 OK on success
    • JSON object representing the updated user
    • 404 Not Found if user with the specified ID is not found

Delete User

  • URL: /api/users/:id
  • Method: DELETE
  • Description: Deletes an existing user.
  • URL Parameters: id: The ID of the user to delete.
  • Response:
    • 200 OK on success
    • 404 Not Found if user with the specified ID is not found
